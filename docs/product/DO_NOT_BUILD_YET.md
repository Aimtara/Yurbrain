# Yurbrain MVP Guardrails: Do Not Build Yet

Yurbrain is a continuity engine for the mind. The MVP must prove the calm loop before expanding into heavier systems:

Capture → Resurface → Continue → Convert → Act → Return

## Product guardrails

- Focus Feed is the home.
- Explore is an optional prototype mode, not the first home screen.
- BrainItem remains the source of truth.
- AI outputs are ItemArtifacts or ThreadMessages, not silent mutations.
- Comments are a core continuation primitive.
- Execution is optional and downstream.
- Not everything important is a task.
- Capture must never require tags, folders, due dates, projects, or categorization.
- Every resurfaced card must explain why it appeared.
- The product should feel calm, personal, non-punitive, and emotionally safe.

## Do not build in the MVP

- Full spatial persistent canvas.
- Persistent ExploreBoard schema.
- Complex graph editor.
- Public remixing or collaboration.
- Social feeds.
- Multiplayer canvas.
- Project hierarchy as a top-level primitive.
- Kanban boards.
- Inbox-zero workflows.
- Required capture triage.
- Required tags/folders/due dates at capture.
- Automatic taskification of every capture.
- Full calendar sync.
- Time block planning.
- Advanced analytics dashboards.
- User-facing productivity scoring.
- Behavioral scoring.
- Gamified points, streak pressure, or leaderboards.
- AI-generated media.
- Multi-agent orchestration.
- Verbose global chatbot interface.
- First-class clusters as runtime objects.
- Separate Explore-only object universe.

## Explore-specific deferrals

Prototype now:

- FeedCard → Explore with this.
- Add 2–5 existing cards.
- Choose Pattern / Idea / Plan / Question.
- Preview a source-linked connection.
- Save Connection Card as an artifact/feed card.
- Return to Focus.

Defer:

- Infinite canvas persistence.
- Saved board management.
- Edge editing/graph operations.
- Complex drag physics.
- Lasso selection.
- Auto-layout engines.
- Collaboration.
- Public sharing/remixing.

## Stop condition

If a proposed change makes Yurbrain feel like a dashboard, task manager, kanban board, inbox, social product, or generic AI chatbot, stop and simplify back to the core loop.
